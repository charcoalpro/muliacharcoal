# Sitewide Content Audit & Targeted Rewrite — Index

**Scope:** 32 content pages (3 guide articles, 5 pillar indexes, all factory / logistics / quality / packaging cluster pages). Pure-utility, legal, product-template and product-listing pages excluded.

**Method:** each page audited against a 10-rule SEO/GEO framework (headings-as-questions, featured-snippet leads, paragraph-intent mapping, anti-bloat, section purity, structure, Devil's Advocate, quantified evidence, mini-cases, ontological/Genesis-Taxonomy-Pragmatics completeness), then **adversarially fact-checked** by an independent agent. Hard guardrails: strict-facts (no invented numbers/cases), respect each page's deliberate omissions, keep charcoal-correct test standards (ASTM D1762 / ISO 18125), treat English-only + per-page FAQPage as by-design.

**Result:** all 32 reports are **fabrication-clean** (verified). Two Devil's-Advocate drafts that smuggled in off-page claims were caught by the fact-checker and corrected by hand (`guide-compare`, `logistics-un-1361`); three minor quote/wording slips fixed (`quality-specifications-explained` binder wording, `logistics-import-uk` HS-code narrative, `packaging-master-box` grams rendering). Mean framework score **≈ 88/100** — the content was already strong; these are refinements, not corrections.

## Cross-cutting patterns

1. **Missing Devil's Advocate sections** — the most common rule-7 FAIL. 18 pages prove a thesis but never give the opposing view its own rebutted H2. A complete, fact-bounded draft is in each report's §4.
2. **Statement-style H2s** that could be buyer questions (rule 1) — near-universal, but low-impact: many H2 IDs are anchor-locked and the FAQ blocks already carry the question intent. Question-form rewrites supplied where they don't break anchors.
3. **Mild restatement / meta-openers** (rule 4) — a handful of "It is worth being precise…" / "Honesty first:" / "Quality is built in…" leads that bury the snippet answer. Compressions supplied.

## How to apply

Each report's **§3 Targeted rewrites** gives paste-ready text keyed to the exact i18n JSON path (e.g. `en.guide.compare.sections.whyCoconut.h2`). **§4** is the drop-in Devil's-Advocate block. Applying any of these means editing `src/i18n/en/*.json` (prose) and, for new sections, adding a small render block to the page's `.astro`. No company facts move out of `src/config/company.ts`.
